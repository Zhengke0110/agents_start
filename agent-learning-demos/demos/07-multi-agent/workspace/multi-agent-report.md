## 写作者（Writer）的报告

# Agent 基础概念与工具系统

## 摘要

Agent 是一种能够自主感知环境、做出推理决策并执行行动的智能程序。它通过融合大语言模型（LLM）的推理能力与外部工具的执行能力，突破了传统 LLM 仅能生成文本的局限。本报告系统梳理 Agent 的核心三要素、区别于普通 LLM 的架构特征、ReAct 模式的工作循环、工具的定义与分类、工具 Schema 结构以及工具调用的安全原则，旨在帮助读者建立对 Agent 及其工具系统的完整认知。

## 正文

### 1. Agent 的核心定义与三要素

Agent 被定义为一个能够自主感知环境、做出决策并采取行动的程序。其核心运作依靠三个相互协作的要素：

-   **感知**：通过输入获取环境信息，例如用户查询、传感器数据或来自其他系统的状态反馈。
-   **推理**：利用大语言模型（LLM）对感知到的信息进行分析、逻辑推演，并决定下一步应采取的合理行动。
-   **行动**：执行具体的操作，如调用外部工具、发送指令或与环境交互。

这三者构成一个闭环，使 Agent 能够持续响应变化并完成复杂任务。

### 2. Agent 与普通 LLM 的本质区别

传统 LLM 依赖其训练数据中的知识生成文本，无法获取实时信息或直接执行操作。Agent 则是一个“LLM + 工具 + 决策循环”的组合体：

-   **实时性**：Agent 可通过工具访问最新数据（如股票价格、天气预报）。
-   **执行能力**：Agent 能调用工具执行具体操作，如发送邮件、查询数据库。
-   **闭环决策**：Agent 通过持续的推理—行动—观察循环，动态调整策略，直至任务完成。

### 3. ReAct 模式的运作流程

ReAct（Reasoning + Acting）模式是 Agent 执行任务的核心范式。该模式包含以下循环步骤：

1.  **推理（Reasoning）**：Agent 分析当前状态，决定需要调用哪个工具以获取所需信息。
2.  **行动（Acting）**：Agent 执行所选工具，并获取其结果。
3.  **观察（Observation）**：Agent 分析工具返回的结果，决定下一步行动——继续推理、调用其他工具，或判断任务已完成。

这一循环持续进行，直到 Agent 认为已经收集到所有必要信息并能够生成最终答案。

### 4. 工具的定义与常见类型

工具是 Agent 可以调用的外部函数，为 Agent 提供超出 LLM 自身能力范围的功能。常见工具类型包括：

-   **计算工具**：执行数学运算、数据聚合等。
-   **信息查询工具**：搜索知识库、访问 API 获取实时数据。
-   **文件系统工具**：读取、写入、管理本地或远程文件。
-   **网络工具**：发起 HTTP 请求、网页抓取等。
-   **数据库工具**：执行 SQL 查询、操作数据库记录。

工具将 LLM 的推理能力与外部世界的执行能力连接起来。

### 5. 工具 Schema 结构

工具 Schema 是描述工具的 JSON “说明书”，它确保 LLM 能够理解如何调用每一个工具。一个典型的工具 Schema 包含以下三个关键要素：

-   **name（工具名称）**：唯一标识，用于在代码中引用。
-   **description（功能描述）**：用自然语言详细说明工具的用途，帮助 LLM 判断何时该使用该工具。
-   **parameters（参数）**：定义工具所需的输入参数及其数据类型（如字符串、数字、数组）。

例如：

```json
{
  "name": "get_weather",
  "description": "获取指定城市的当前天气信息",
  "parameters": {
    "city": { "type": "string", "description": "城市名称" }
  }
}
```

### 6. 工具调用的安全原则

为确保 Agent 工具调用的安全性，需要遵循四项基本原则：

-   **最小权限**：只为 Agent 授予完成当前任务所必需的最少权限，避免赋予不必要的高级权限。
-   **路径沙箱**：当涉及文件操作时，限制工具可以访问的文件目录范围，防止越权读写敏感文件。
-   **输入验证**：对模型传入的工具参数进行严格的校验和清理，防止恶意输入导致系统破坏。
-   **用户确认**：对删除、写数据库等高风险操作，要求人工确认后再执行，提供二次防护。

## 总结

Agent 通过将大语言模型的推理能力与外部工具的执行能力相结合，实现了超越文本生成的智能交互。其核心三要素（感知、推理、行动）和 ReAct 模式构成了动态决策循环的基础。工具作为 Agent 与外部世界交互的桥梁，通过严格的 Schema 描述和安全原则确保调用的准确性与安全性。掌握这些基础概念，是理解并构建高效、可靠 Agent 系统的前提。
