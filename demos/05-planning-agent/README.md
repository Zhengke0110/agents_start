# Demo 05：Planning Agent 任务规划

## 学习目标

理解 Agent 如何把复杂任务拆成步骤，用 Todo List 跟踪进度。

## 核心流程

```
用户："帮我准备一次技术分享"
          │
          ▼
   ① 规划阶段：Agent 分析任务，创建 Todo
      - [ ] 1. 准备 PPT 大纲
      - [ ] 2. 撰写演讲稿
      - [ ] 3. 准备 Demo 代码
          │
          ▼
   ② 执行阶段：Agent 逐个执行
      - [>] 1. 准备 PPT 大纲     ← in_progress
         完成 → [X] 1. 准备 PPT 大纲
      - [>] 2. 撰写演讲稿        ← in_progress
         完成 → [X] 2. 撰写演讲稿
      - [>] 3. 准备 Demo 代码    ← in_progress
         完成 → [X] 3. 准备 Demo 代码
          │
          ▼
   ③ 报告：全部完成！
```

## 任务状态机

```
pending ──→ in_progress ──→ completed
 待办         进行中         已完成
```

## 本 Demo 的工具

| 工具         | 功能         |
| ------------ | ------------ |
| `createTodo` | 创建新任务   |
| `updateTodo` | 更新任务状态 |
| `listTodos`  | 列出所有任务 |

## 关键设计

**system prompt 中的规则**：

> 一次只做一个任务（in_progress），完成后标记 completed 再做下一个。

这模拟了真实的项目执行方式——不并行、不跳跃、有序推进。

## 运行

```bash
pnpm run demo:05
```

## 规划对 Agent 的作用

1. **结构化**：把模糊的大任务变成清晰的小步骤
2. **可追踪**：知道当前做到哪了、还剩什么
3. **防遗漏**：不会漏掉步骤
4. **可恢复**：中断后可以从 Todo List 继续

## 规划失败的可能问题

- **计划太细**：10 步的活列了 50 个 todo，浪费 token
- **计划太粗**：步骤描述不清，执行时不知道做什么
- **顺序错误**：依赖关系没考虑，比如"写报告"排在"查资料"前面
- **忘记更新状态**：Agent 做了事但没标记 completed，一直 in_progress
